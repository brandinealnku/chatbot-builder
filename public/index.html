Architecture

  • Firebase Auth handles email/password signup and sign-in.
  • Firestore stores owner profiles in users/{uid}.
  • Private chatbots live in users/{uid}/chatbots.
  • Public widget-safe mirrors live in publicChatbots/{chatbotId}.
  • Public widget-submitted leads land in publicLeads.
  • Owners can mirror leads into users/{uid}/leads.
  • Optional Cloud Function handles secure AI chat replies.

Why this is better than pure client-side

Firebase gives you a realistic path to production without building your own backend first. Firestore supports direct web access, document writes, and query reads from the browser, while Functions gives you a place for secure server-only logic.
